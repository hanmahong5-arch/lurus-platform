# DNS Records for mail.lurus.cn
# DNS 记录配置 — 邮件认证强化

All records should be configured in your DNS provider for `lurus.cn`.

## MX Record

```
Type: MX
Host: @
Value: mail.lurus.cn
Priority: 10
TTL: 3600
```

## SPF

```
Type: TXT
Host: @
Value: v=spf1 mx a:mail.lurus.cn include:sendcloud.net ~all
TTL: 3600
```

Note: Start with `~all` (softfail). After 1 week of monitoring, upgrade to `-all` (hardfail).

## DKIM

DKIM key is auto-generated by Stalwart. Retrieve the public key:

```bash
# Via Stalwart Admin API
ssh root@100.98.57.55 "kubectl exec -n mail stalwart-0 -- /bin/sh -c 'cat /opt/stalwart-mail/data/dkim/*.pub'"

# Or via Admin UI at https://mail.lurus.cn/admin -> Settings -> DKIM
```

Then add a TXT record:

```
Type: TXT
Host: default._domainkey
Value: v=DKIM1; k=rsa; p=<PUBLIC_KEY_BASE64>
TTL: 3600
```

## DMARC

### Phase 1 (monitoring — first 2 weeks):

```
Type: TXT
Host: _dmarc
Value: v=DMARC1; p=none; rua=mailto:dmarc-reports@lurus.cn; ruf=mailto:dmarc-reports@lurus.cn; sp=none; adkim=r; aspf=r; pct=100; ri=86400
TTL: 3600
```

### Phase 2 (quarantine — after 2 weeks clean reports):

```
Type: TXT
Host: _dmarc
Value: v=DMARC1; p=quarantine; rua=mailto:dmarc-reports@lurus.cn; ruf=mailto:dmarc-reports@lurus.cn; sp=quarantine; adkim=s; aspf=s; pct=100; ri=86400
TTL: 3600
```

### Phase 3 (reject — after 4 weeks total):

```
Type: TXT
Host: _dmarc
Value: v=DMARC1; p=reject; rua=mailto:dmarc-reports@lurus.cn; ruf=mailto:dmarc-reports@lurus.cn; sp=reject; adkim=s; aspf=s; pct=100; ri=86400
TTL: 3600
```

## MTA-STS

Create `_mta-sts` TXT record and host the policy file:

```
Type: TXT
Host: _mta-sts
Value: v=STSv1; id=20260227
TTL: 3600
```

Policy file at `https://mta-sts.lurus.cn/.well-known/mta-sts.txt`:

```
version: STSv1
mode: enforce
mx: mail.lurus.cn
max_age: 604800
```

## SMTP TLS Reporting

```
Type: TXT
Host: _smtp._tls
Value: v=TLSRPTv1; rua=mailto:tls-reports@lurus.cn
TTL: 3600
```

## Reverse DNS (PTR)

Ensure the server IP has a PTR record pointing to `mail.lurus.cn`.
This must be configured at the hosting/VPS provider level.

## SendCloud SPF (if using SendCloud relay)

Add SendCloud's SPF to your domain:

```
Type: TXT
Host: @
Value: v=spf1 mx a:mail.lurus.cn include:sendcloud.net ~all
TTL: 3600
```

(Already included in the main SPF record above.)
